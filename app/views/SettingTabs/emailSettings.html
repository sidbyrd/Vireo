TEMPLATES VARIABLES

Variables can be added to the email templates, both in the subject line and in the body.Variable names are will be all uppercase, and should be surrounded by braces if you want them to be automatically replaced with the associated values. The available values are:

FULL_NAME: The student's full, official name.
FIRST_NAME: The student's first name.
LAST_NAME: The student's last name.
STUDENT_ID: The student's student ID.
STUDENT_NETID: The student's NetID.
STUDENT_EMAIL: The student's email address.
DOCUMENT_TITLE: The title of the thesis or dissertation as supplied by the student.
DOCUMENT_TYPE: The document type, typically 'Thesis' or 'Dissertation'. This is defined by your office.
GRAD_SEMESTER: The semester in which the student indicated they will graduate.
STUDENT_URL: The URL that will allow the student to view or take action on their submission.
ADVISOR_URL: The URL sent to the student's advisor to request manuscript approval.
SUBMISSION_ID: The ID number used by Vireo to keep track of the submission
SUBMISSION_STATUS: The current status of the submission; e.g. 'Approved', 'Needs Corrections', etc.
SUBMISSION_ASSIGNED_TO: The name of the staff member to which this submission is currently assigned.
REGISTRATION_URL: The URL back to Vireo to validate ownership of an email address. (This variable is only available in select SYSTEM templates)

When these variables are included in a template, the text will be replaced prior to the email being sent (or the comment being recorded in the action log). For example, the string

Dear {FIRST_NAME} {LAST_NAME}:

would be replaced with:

Dear John Doe:

 

Not all variables will have a value all of the time, due to missing information or just changing status. For example, if the submission is not assigned to anyone, then SUBMISSION_ASSIGNED_TO will be blank. You can use || to chain several variables together in one set of braces, like {VARIABLE_NAME||ANOTHER_VARIABLE||default text}, and the first variable that isn't blank will replace the entire group. At the end, you can put some default text that will appear if all the other variables are blank. For example, the string

Assigned to: {SUBMISSION_ASSIGNED_TO||n/a}

might be replaced with

Assigned to: Einstein, Albert if it's currently assigned, or Assigned to: n/a if not.
